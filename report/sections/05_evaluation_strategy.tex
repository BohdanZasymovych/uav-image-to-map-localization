\section{Evaluation Strategy}

\subsection{Evaluation Protocol}
To ensure controlled and reproducible validation, we evaluate the pipeline on a synthetic dataset generated from a high-resolution satellite orthophoto. Each test frame is created by sampling a map patch and applying an affine deformation around its center. The transformed patch is treated as a synthetic UAV frame, while the center of the original sampled patch is used as ground truth in map pixel coordinates.

For evaluation, we generate $N$ frames and run the full localization pipeline independently on each frame. For frame $i$, we obtain estimated map coordinates $(\hat{X}_i, \hat{Y}_i)$, runtime $t_i$, and matching statistics $(n_i^{\text{inliers}}, n_i^{\text{matches}})$. Frames that fail due to insufficient correspondences or model estimation errors are skipped and excluded from aggregate statistics.

\subsection{Evaluated Metrics}
The primary accuracy metric is Root Mean Square Error (RMSE), which measures the Euclidean localization error in map pixels:
\begin{equation}
e_i = \sqrt{(X_i - \hat{X}_i)^2 + (Y_i - \hat{Y}_i)^2},
\end{equation}
\begin{equation}
RMSE = \sqrt{\frac{1}{N}\sum_{i=1}^{N} e_i^2}.
\end{equation}

To characterize efficiency, we report runtime mean and standard deviation:
\begin{equation}
\mu_t = \frac{1}{N}\sum_{i=1}^{N} t_i, \qquad
\sigma_t = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(t_i - \mu_t)^2}.
\end{equation}

To characterize geometric consistency of matching, we use the mean inlier ratio:
\begin{equation}
r_i = \frac{n_i^{\text{inliers}}}{n_i^{\text{matches}}}, \qquad
\bar{r} = \frac{1}{N}\sum_{i=1}^{N} r_i,
\end{equation}
with $r_i=0$ when $n_i^{\text{matches}}=0$.

Together, these metrics provide complementary evidence: RMSE captures localization accuracy, runtime statistics reflect computational cost, and inlier ratio indicates robustness of correspondence filtering.
