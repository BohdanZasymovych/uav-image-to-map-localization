\section{Results}\label{sec:results}

At the current stage, we have implemented a Command Line Interface (CLI) application that ingests a reference map and a UAV camera image, matches their features, and calculates the absolute map coordinates of the UAV image's center. The script is driven by a configuration file that defines the geometric transformation model and the necessary algorithmic parameters. All code is available on the our \href{https://github.com/BohdanZasymovych/uav-image-to-map-localization.git}{GitHub repository}

\subsection{Single Image Match}\label{subsec:single_image_match}
A single-image localization test was executed using the Affine transformation model. The inputs consisted of a reference satellite map and a single simulated UAV frame from our artificially generated dataset. The run produced the following metrics:

\begin{itemize}
    \item \textbf{Raw matches:} 478
    \item \textbf{Inliers after RANSAC:} 471
    \item \textbf{Runtime (without image rendering):} 1.045 s
    \item \textbf{Estimated map position:} $(179.99, 866.84)$ px
\end{itemize}

\subsection{Dataset Evaluation}\label{subsec:dataset_evaluation}
To quantify pipeline behavior beyond a single frame, we ran the dataset-level evaluation on 60 synthetically generated UAV crops over the same reference map. The evaluator successfully localized 54 frames and skipped 6 frames due to lack of the found features after RANSAC filtering. The aggregate results are:

\begin{itemize}
    \item \textbf{Successful localizations:} 54 / 60 (90\%)
    \item \textbf{RMSE:} 16.69 px
    \item \textbf{Mean runtime:} $0.459 \pm 0.228$ s
    \item \textbf{Mean inlier ratio:} 0.807
\end{itemize}
